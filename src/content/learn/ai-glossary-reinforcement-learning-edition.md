Reinforcement learning (RL) has its own vocabulary. Here's what the terms actually mean.

## Core Concepts

**Agent** — the learner or decision-maker. In a game, it's the player. In robotics, it's the robot. In RLHF, it's the language model being fine-tuned.

**Environment** — everything the agent interacts with. The game world, the physical world, or (in RLHF) the prompt-response evaluation loop. The agent acts, the environment responds.

**State** — a snapshot of the environment at a given moment. In chess, the board position. In a dialogue system, the conversation history so far. The state contains everything the agent needs to decide its next action.

**Action** — what the agent can do. Move a piece, generate a token, adjust a thermostat. The **action space** is the set of all possible actions — it can be discrete (left/right/up/down) or continuous (torque values for a robotic arm).

**Reward** — a numerical signal from the environment telling the agent how good its action was. Positive reward = good, negative = bad. The agent's entire goal is to maximize cumulative reward over time.

**Policy (π)** — the agent's strategy: a mapping from states to actions. A policy can be deterministic (always take action A in state S) or stochastic (take action A with probability 0.7 in state S). Training an RL agent means finding a good policy.

**Value Function V(s)** — the expected total future reward starting from state s, following the current policy. It answers: "How good is it to be in this state?" States with high value lead to high cumulative rewards.

**Q-Function Q(s, a)** — the expected total future reward from taking action a in state s, then following the current policy. It answers: "How good is this specific action in this specific state?"

**Episode** — one complete sequence from start to end. A game from beginning to game-over. A conversation from first message to completion. Not all RL problems have natural episodes (called **continuing tasks**).

**Discount Factor (γ)** — a number between 0 and 1 that controls how much the agent values future vs. immediate rewards. γ = 0 means "only care about immediate reward." γ = 0.99 means "future rewards are almost as important as immediate ones."

## Learning Methods

**Model-Free RL** — the agent learns directly from experience without building a model of the environment. Most practical RL is model-free. Methods like Q-learning and policy gradient are model-free.

**Model-Based RL** — the agent learns a model of the environment (predicting next state and reward given current state and action) and uses it to plan. More sample-efficient but harder to get right. AlphaZero is a famous example.

**On-Policy** — learning from data generated by the current policy. The agent collects experience, learns from it, then discards it and collects new experience with the updated policy. Example: PPO.

**Off-Policy** — learning from data generated by any policy, including old versions of itself or even random exploration. More sample-efficient because you can reuse old data. Example: DQN, SAC.

**Q-Learning** — learns the Q-function directly. The agent stores Q-values for state-action pairs and updates them toward the target: reward + γ × max Q(next state, any action). When the Q-function is accurate, the optimal policy is: pick the action with highest Q-value.

**Deep Q-Network (DQN)** — Q-learning with a neural network approximating the Q-function. Breakthrough: DeepMind's Atari game player (2013). Key innovations: experience replay (storing and resampling past transitions) and target networks (using a slowly-updated copy of the Q-network for stable targets).

**Policy Gradient** — directly optimizes the policy by computing gradients of expected reward with respect to policy parameters. The REINFORCE algorithm is the simplest version: increase probability of actions that led to high reward, decrease probability of actions that didn't.

**Actor-Critic** — combines policy gradient (actor) with a value function (critic). The actor decides what to do; the critic evaluates how good the decision was. The critic's evaluation reduces variance in the policy gradient, making learning more stable.

**Proximal Policy Optimization (PPO)** — the most widely used policy gradient algorithm. Prevents the policy from changing too drastically in a single update by clipping the objective function. If you see RL in a modern ML pipeline, it's probably PPO.

**Soft Actor-Critic (SAC)** — an off-policy actor-critic method that adds an entropy bonus to the reward, encouraging exploration. The agent tries to maximize reward while being as "random" as possible — this prevents premature convergence and improves robustness.

## RL for Language Models

**RLHF (Reinforcement Learning from Human Feedback)** — the process of fine-tuning a language model using human preferences as the reward signal. Steps: (1) collect human comparisons of model outputs, (2) train a reward model on those comparisons, (3) use RL (usually PPO) to optimize the language model against the reward model.

**Reward Model** — a model trained to predict which of two outputs a human would prefer. It converts subjective human judgment into a scalar reward signal that RL algorithms can optimize. Quality of the reward model is the bottleneck for RLHF.

**KL Penalty** — a constraint that prevents the RL-trained model from drifting too far from the original supervised model. Without it, the model can "hack" the reward model — finding outputs that score high on the reward model but are actually low quality (reward hacking).

**DPO (Direct Preference Optimization)** — an alternative to RLHF that skips the reward model entirely. It directly optimizes the policy using preference pairs, reformulating the RL problem as a classification problem. Simpler to implement, increasingly popular.

**GRPO (Group Relative Policy Optimization)** — generates multiple responses per prompt, scores them, and uses the relative rankings to update the policy. No separate reward model needed. Used in DeepSeek-R1 and gaining traction.

## Exploration vs. Exploitation

**Exploration** — trying new, potentially suboptimal actions to discover better strategies. Without exploration, the agent gets stuck with the first decent strategy it finds.

**Exploitation** — using the best-known strategy to maximize reward. Pure exploitation misses better options.

**Epsilon-Greedy** — the simplest exploration strategy. With probability ε, take a random action; otherwise, take the best-known action. Start with high ε (lots of exploration) and decay over time.

**Curiosity-Driven Exploration** — reward the agent for reaching states that are "surprising" (hard to predict). This drives exploration without random action selection.

## Multi-Agent RL

**Multi-Agent RL (MARL)** — RL with multiple agents interacting in the same environment. They might cooperate (team sports), compete (board games), or do both (traffic). The challenge: each agent's optimal strategy depends on what other agents do, creating a moving target.

**Self-Play** — training an agent by having it play against copies of itself. AlphaGo and AlphaZero used self-play to achieve superhuman performance. The agent continuously creates its own curriculum as it improves.
